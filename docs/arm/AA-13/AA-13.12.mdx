<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} [A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions expresses the user's intent to abide by certain restrictions. A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile expresses the user's intent to abide by a set of Restrictions or other specified run-time policies. These may facilitate the construction of simpler run-time environments.] </p>

#### Syntax

<p>The form of a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions is as follows: </p>
<p>  <strong>pragma</strong> Restrictions(<code><a href="../AA-13/AA-13.12#S0359">restriction</a></code>{"{"}, <code><a href="../AA-13/AA-13.12#S0359">restriction</a></code>{"}"});</p>
<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} <code>restriction</code><a id="S0359"></a><code> ::= </code><em>restriction_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>    | <em>restriction_parameter_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> ={">"} <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code></p>
<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} <code>restriction_parameter_argument</code><a id="S0360"></a><code> ::= </code><code><a href="../AA-4/AA-4.1#S0091">name</a></code> | <code><a href="../AA-4/AA-4.4#S0132">expression</a></code></p>

#### Name Resolution Rules

<p>Unless otherwise specified for a particular restriction, the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> is expected to be of any integer type. </p>

#### Legality Rules

<p>Unless otherwise specified for a particular restriction, the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> shall be static, and its value shall be nonnegative. </p>

#### Post-Compilation Rules

<p>{"{"}<em>{"AI05-0013-1"}</em>{"}"} A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions is a configuration pragma. If a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions applies to any compilation unit included in the partition, this may impose either (or both) of two kinds of requirements, as specified for the particular restriction:</p>
<p>{"{"}<em>{"AI05-0013-1"}</em>{"}"} A restriction may impose requirements on some or all of the units comprising the partition. Unless otherwise specified for a particular restriction, such a requirement applies to all of the units comprising the partition and is enforced via a post-compilation check.</p>
<p>{"{"}<em>{"AI05-0013-1"}</em>{"}"} A restriction may impose requirements on the run-time behavior of the program, as indicated by the specification of run-time behavior associated with a violation of the requirement.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>In this latter case, there is no post-compilation check needed for the requirement. </p></Admonition>

<p>{"{"}<em>8652/0042</em>{"}"} {"{"}<em>{"AI95-00130-01"}</em>{"}"} For the purpose of checking whether a partition contains constructs that violate any restriction (unless specified otherwise for a particular restriction):</p>
<p>{"{"}<em>8652/0042</em>{"}"} {"{"}<em>{"AI95-00130-01"}</em>{"}"} Generic instances are logically expanded at the point of instantiation;</p>
<p>{"{"}<em>8652/0042</em>{"}"} {"{"}<em>{"AI95-00130-01"}</em>{"}"} If an object of a type is declared or allocated and not explicitly initialized, then all expressions appearing in the definition for the type and any of its ancestors are presumed to be used;</p>
<p>{"{"}<em>8652/0042</em>{"}"} {"{"}<em>{"AI95-00130-01"}</em>{"}"} A <code><a href="../AA-3/AA-3.7#S0063">default_expression</a></code> for a formal parameter or a generic formal object is considered to be used if and only if the corresponding actual parameter is not provided in a given call or instantiation. </p>

#### Implementation Permissions

<p>{"{"}<em>{"AI05-0269-1"}</em>{"}"} An implementation may provide implementation-defined restrictions; the identifier for an implementation-defined restriction shall differ from those of the language-defined restrictions. </p>
<Admonition type="note">
<p><strong>Implementation defined: </strong>Implementation-defined restrictions allowed in a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions.</p></Admonition>

<p>An implementation may place limitations on the values of the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> that are supported, and limitations on the supported combinations of restrictions. The consequences of violating such limitations are implementation defined. </p>
<Admonition type="note">
<p><strong>Implementation defined: </strong>The consequences of violating limitations on Restrictions <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s.</p></Admonition>

<Admonition type="note">
<p><strong>Ramification: </strong>Such limitations may be enforced at compile time or at run time. Alternatively, the implementation is allowed to declare violations of the restrictions to be erroneous, and not enforce them at all. </p></Admonition>

<p>{"{"}<em>8652/0042</em>{"}"} {"{"}<em>{"AI95-00130-01"}</em>{"}"} An implementation is permitted to omit restriction checks for code that is recognized at compile time to be unreachable and for which no code is generated.</p>
<p>{"{"}<em>8652/0043</em>{"}"} {"{"}<em>{"AI95-00190-01"}</em>{"}"} Whenever enforcement of a restriction is not required prior to execution, an implementation may nevertheless enforce the restriction prior to execution of a partition to which the restriction applies, provided that every execution of the partition would violate the restriction. </p>

#### Syntax

<p>{"{"}<em>{"AI95-00249-01"}</em>{"}"} {"{"}<em>{"AI05-0246-1"}</em>{"}"} The form of a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile is as follows: </p>
<p>  <strong>pragma</strong> Profile (<em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> {"{"}, <em>profile_</em><code><a href="../AA-2/AA-2.8#S0020">pragma_argument_association</a></code>{"}"});</p>

#### Legality Rules

<p>{"{"}<em>{"AI95-00249-01"}</em>{"}"} {"{"}<em>{"AI05-0246-1"}</em>{"}"} The <em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> shall be the name of a usage profile. The semantics of any <em>profile_</em><code><a href="../AA-2/AA-2.8#S0020">pragma_argument_association</a></code>s are defined by the usage profile specified by the <em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. </p>

#### Static Semantics

<p>{"{"}<em>{"AI95-00249-01"}</em>{"}"} {"{"}<em>{"AI05-0246-1"}</em>{"}"} A profile is equivalent to the set of configuration pragmas that is defined for each usage profile. </p>

#### Post-Compilation Rules

<p>{"{"}<em>{"AI95-00249-01"}</em>{"}"} A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile is a configuration pragma. There may be more than one <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile for a partition. </p>

#### Implementation Permissions

<p>{"{"}<em>{"AI05-0269-1"}</em>{"}"} An implementation may provide implementation-defined usage profiles; the identifier for an implementation-defined usage profile shall differ from those of the language-defined usage profiles.</p>
<Admonition type="note">
<p><strong>Implementation defined: </strong>Implementation-defined usage profiles allowed in a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile.</p></Admonition>

<Admonition type="note">
<p>NOTE 1   {"{"}<em>{"AI95-00347-01"}</em>{"}"} Restrictions intended to facilitate the construction of efficient tasking run-time systems are defined in D.7. Restrictions intended for use when constructing high integrity systems are defined in H.4.</p></Admonition>

<Admonition type="note">
<p>NOTE 2   An implementation has to enforce the restrictions in cases where enforcement is required, even if it chooses not to take advantage of the restrictions in terms of efficiency. </p></Admonition>

<Admonition type="note">
<p><strong>Discussion: </strong>It is not the intent that an implementation will support a different run-time system for every possible combination of restrictions. An implementation might support only two run-time systems, and document a set of restrictions that is sufficient to allow use of the more efficient and safe one. </p></Admonition>

## 13.12.1  Language-Defined Restrictions and Profiles


#### Static Semantics

<p>{"{"}<em>{"AI95-00257-01"}</em>{"}"} The following <em>restriction_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>s are language defined (additional restrictions are defined in the Specialized Needs Annexes):</p>
<p>{"{"}<em>{"AI05-0241-1"}</em>{"}"} No_Implementation_Aspect_Specifications There are no implementation-defined aspects specified by an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>. This restriction applies only to the current compilation or environment, not the entire partition. </p>
<Admonition type="note">
<p><strong>Discussion: </strong>{"{"}<em>{"AI05-0241-1"}</em>{"}"} This restriction (as well as others below) applies only to the current compilation, because it is likely that the runtime (and possibly user-written low-level code) will need to use implementation-defined aspects. But a partition-wide restriction applies everywhere, including the runtime. </p></Admonition>

<p>{"{"}<em>{"AI95-00257-01"}</em>{"}"} No_Implementation_Attributes There are no implementation-defined attributes. This restriction applies only to the current compilation or environment, not the entire partition. </p>
<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} {"{"}<em>{"AI05-0269-1"}</em>{"}"} No_Implementation_Identifiers There are no usage names that denote declarations with implementation-defined identifiers that occur within language-defined packages or instances of language-defined generic packages. Such identifiers can arise as follows: </p>
<p>The following language-defined packages and generic packages allow implementation-defined identifiers:</p>
<p>package System (see 13.7);</p>
<p>package Standard (see A.1);</p>
<p>package Ada.Command_Line (see A.15);</p>
<p>package Interfaces.C (see B.3);</p>
<p>package Interfaces.C.Strings (see B.3.1);</p>
<p>package Interfaces.C.Pointers (see B.3.2);</p>
<p>package Interfaces.COBOL (see B.4);</p>
<p>package Interfaces.Fortran (see B.5); </p>
<p>The following language-defined packages contain only implementation-defined identifiers:</p>
<p>package System.Machine_Code (see 13.8);</p>
<p>package Ada.Directories.Information (see A.16);</p>
<p>nested Implementation packages of the Queue containers (see A.18.28-31);</p>
<p>package Interfaces (see B.2);</p>
<p>package Ada.Interrupts.Names (see C.3.2).</p>
<p>For package Standard, Standard.Long_Integer and Standard.Long_Float are considered language-defined identifiers, but identifiers such as Standard.Short_Short_Integer are considered implementation-defined.</p>
<p>This restriction applies only to the current compilation or environment, not the entire partition.</p>
<p>{"{"}<em>{"AI95-00257-01"}</em>{"}"} No_Implementation_Pragmas There are no implementation-defined pragmas or pragma arguments. This restriction applies only to the current compilation or environment, not the entire partition.</p>
<p>{"{"}<em>{"AI05-0242-1"}</em>{"}"} No_Implementation_Units There is no mention in the <code><a href="../AA-10/AA-10.1#S0292">context_clause</a></code> of any implementation-defined descendants of packages Ada, Interfaces, or System. This restriction applies only to the current compilation or environment, not the entire partition.</p>
<p>{"{"}<em>{"AI95-00368-01"}</em>{"}"} {"{"}<em>{"AI05-0229-1"}</em>{"}"} No_Obsolescent_Features There is no use of language features defined in Annex J. It is implementation defined whether uses of the renamings of J.1 and of the <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s of J.15 are detected by this restriction. This restriction applies only to the current compilation or environment, not the entire partition. </p>
<Admonition type="note">
<p><strong>Reason: </strong>A user could compile a rename like </p></Admonition>

<CodeBlock>
with Ada.Text_IO;{"\n"}package Text_IO renames Ada.Text_IO;{"\n"}
</CodeBlock>

<Admonition type="note">
<p>Such a rename must not be disallowed by this restriction, nor should the compilation of such a rename be restricted by an implementation. Many implementations implement the renames of J.1 by compiling them normally; we do not want to require implementations to use a special mechanism to implement these renames.</p></Admonition>

<Admonition type="note">
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} The pragmas have the same functionality as the corresponding aspect (unlike the typical obsolescent feature), and rejecting them could be a significant portability problem for existing code. </p></Admonition>

<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} {"{"}<em>{"AI05-0241-1"}</em>{"}"} The following <em>restriction_parameter_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>s are language defined:</p>
<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} No_Dependence Specifies a library unit on which there are no semantic dependences.</p>
<p>{"{"}<em>{"AI05-0241-1"}</em>{"}"} No_Specification_of_Aspect Identifies an aspect for which no <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>, <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>, or <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> is given.</p>
<p>{"{"}<em>{"AI05-0272-1"}</em>{"}"} No_Use_Of_Attribute Identifies an attribute for which no <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code> or <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code> is given.</p>
<p>{"{"}<em>{"AI05-0272-1"}</em>{"}"} No_Use_Of_Pragma Identifies a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> which is not to be used.</p>
<p>{"{"}<em>{"AI12-0389-1"}</em>{"}"} No_Unrecognized_Aspects There are no <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>s having an unrecognized <em>aspect_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. This restriction applies only to the current compilation or environment, not the entire partition.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>When this restriction is in effect, unrecognized aspects cannot be ignored in the current compilation; they violate the restriction. This is true despite the Implementation Permissions of 13.1.1. </p></Admonition>

<p>{"{"}<em>{"AI12-0389-1"}</em>{"}"} No_Unrecognized_Pragmas There are no <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s having an unrecognized pragma <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. This restriction applies only to the current compilation or environment, not the entire partition.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>When this restriction is in effect, unrecognized pragmas cannot be ignored in the current compilation; they violate the restriction. This is true despite the rules of 2.8. </p></Admonition>


#### Legality Rules

<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} {"{"}<em>{"AI12-0444-1"}</em>{"}"} The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Dependence restriction shall be a <code><a href="../AA-4/AA-4.1#S0091">name</a></code>; the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> shall have the form of a full expanded name of a library unit, but can be a name that has no corresponding unit currently present in the environment.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>This <code><a href="../AA-4/AA-4.1#S0091">name</a></code> is not resolved. </p></Admonition>

<p>{"{"}<em>{"AI05-0241-1"}</em>{"}"} The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Specification_of_Aspect restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>; this is an identifier specific to a pragma (see 2.8) and does not denote any declaration.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. As for No_Dependence, there is no check that the aspect <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> is meaningful; it might refer to an implementation-defined aspect on one implementation, but nothing at all on another implementation. </p></Admonition>

<p>{"{"}<em>{"AI05-0272-1"}</em>{"}"} The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Use_Of_Attribute restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> or one of the reserved words Access, Delta, Digits, Mod, or Range; this is an identifier specific to a pragma.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. There is no check that the attribute identifier refers to a known <code><a href="../AA-4/AA-4.1#S0101">attribute_designator</a></code>; it might refer to an implementation-defined attribute on one implementation, but nothing at all on another implementation. </p></Admonition>

<p>{"{"}<em>{"AI05-0272-1"}</em>{"}"} The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Use_Of_Pragma restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> or the reserved word Interface; this is an identifier specific to a pragma.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. There is no check that the pragma identifier refers to a known <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>; it might refer to an implementation-defined pragma on one implementation, but nothing at all on another implementation. </p></Admonition>


#### Post-Compilation Rules

<p>{"{"}<em>{"AI95-00381-01"}</em>{"}"} {"{"}<em>{"AI05-0241-1"}</em>{"}"} No compilation unit included in the partition shall depend semantically on the library unit identified by the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> of a No_Dependence restriction.</p>
<Admonition type="note">
<p><strong>Ramification: </strong>{"{"}<em>{"AI12-0005-1"}</em>{"}"} There is no requirement that the library unit actually exists. One possible use of the pragma is to prevent the use of implementation-defined units; when the program is ported to a different compiler, it is perfectly reasonable that no unit with the name exists. </p></Admonition>


#### Static Semantics

<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} The following <em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> is language defined:</p>
<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} No_Implementation_Extensions </p>
<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} For usage profile No_Implementation_Extensions, there shall be no <em>profile_</em><code><a href="../AA-2/AA-2.8#S0020">pragma_argument_association</a></code>s.</p>
<p>{"{"}<em>{"AI05-0246-1"}</em>{"}"} The No_Implementation_Extensions usage profile is equivalent to the following restrictions: </p>
<CodeBlock>
No_Implementation_Aspect_Specifications,{"\n"}No_Implementation_Attributes,{"\n"}No_Implementation_Identifiers,{"\n"}No_Implementation_Pragmas,{"\n"}No_Implementation_Units.{"\n"}
</CodeBlock>
