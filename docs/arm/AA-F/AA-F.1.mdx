#### Static Semantics

<AnnotatedOnly>
<MarginText>
1/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0272-1"}</em>{"}"} <br />The representation attribute Machine_Radix may be specified for a decimal first subtype (see <a href="../AA-3/AA-3.5#Subclause_3.5.9">3.5.9</a>) via an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>; the expression of such a clause shall be static, and its value shall be 2 or 10. A value of 2 implies a binary base range; a value of 10 implies a decimal base range. <br /></p>

<AnnotatedOnly>
<MarginText>
1.a
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>In the absence of a Machine_Radix clause, the choice of 2 versus 10 for S'Machine_Radix is not specified. <br /></Admonition>

<AnnotatedOnly>
<MarginText>
1.b/3
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="note" title="Note: ">
<strong>Aspect Description for </strong><strong>Machine_Radix: </strong>Radix (2 or 10) that is used to represent a decimal fixed point type.<br /></Admonition>


#### Implementation Advice

<AnnotatedOnly>
<MarginText>
2
</MarginText>
</AnnotatedOnly>
<p>Packed decimal should be used as the internal representation for objects of subtype S when S'Machine_Radix = 10. <br /></p>

<AnnotatedOnly>
<MarginText>
2.a/2
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="implementation-advice" title="Implementation Advice">
<strong></strong>Packed decimal should be used as the internal representation for objects of subtype <em>S</em> when <em>S</em>'Machine_Radix = 10.<br /></Admonition>

<AnnotatedOnly>
<MarginText>
2.b/3
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>{"{"}<em>{"AI05-0229-1"}</em>{"}"} <br />The intent of a decimal Machine_Radix attribute definition clause is to allow the programmer to declare an Ada decimal data object whose representation matches a particular COBOL implementation's representation of packed decimal items. The Ada object may then be passed to an interfaced COBOL program that takes a packed decimal data item as a parameter, assuming that convention COBOL has been specified for the Ada object's type with an aspect Convention.<br /></Admonition>

<AnnotatedOnly>
<MarginText>
2.c
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="note" title="Note: ">
Additionally, the Ada compiler may choose to generate arithmetic instructions that exploit the packed decimal representation.<br /></Admonition>
