# 5.7  Exit Statements

<p>[An <code><a href="./AA-5.7#S0193">exit_statement</a></code> is used to complete the execution of an enclosing <code><a href="./AA-5.5#S0178">loop_statement</a></code>; the completion is conditional if the <code><a href="./AA-5.7#S0193">exit_statement</a></code> includes a <code><a href="./AA-4.5#S0150">condition</a></code>.] </p>

#### Syntax

<p><code>exit_statement</code><a id="S0193"></a><code> ::= </code>   <strong>exit</strong> [<em>loop_</em><code><a href="./AA-4.1#S0091">name</a></code>] [<strong>when</strong> <code><a href="./AA-4.5#S0150">condition</a></code>];</p>

#### Name Resolution Rules

<p>The <em>loop_</em><code><a href="./AA-4.1#S0091">name</a></code>, if any, in an <code><a href="./AA-5.7#S0193">exit_statement</a></code> shall resolve to denote a <code><a href="./AA-5.5#S0178">loop_statement</a></code>. </p>

#### Legality Rules

<p>Each <code>[exit_statement](./AA-5.7#S0193)</code> <em>applies to</em> a <code>[loop_statement](./AA-5.5#S0178)</code>; this is the <code>[loop_statement](./AA-5.5#S0178)</code> being exited. An <code>[exit_statement](./AA-5.7#S0193)</code> with a <code><a href="./AA-4.1#S0091">name</a></code> is only allowed within the <code>[loop_statement](./AA-5.5#S0178)</code> denoted by the <code><a href="./AA-4.1#S0091">name</a></code>, and applies to that <code>[loop_statement](./AA-5.5#S0178)</code>. An <code>[exit_statement](./AA-5.7#S0193)</code> without a <code><a href="./AA-4.1#S0091">name</a></code> is only allowed within a <code>[loop_statement](./AA-5.5#S0178)</code>, and applies to the innermost enclosing one. An <code>[exit_statement](./AA-5.7#S0193)</code> that applies to a given <code>[loop_statement](./AA-5.5#S0178)</code> shall not appear within a body or <code>[accept_statement](./AA-9.5#S0258)</code>, if this construct is itself enclosed by the given <code><a href="./AA-5.5#S0178">loop_statement</a></code>. </p>

#### Dynamic Semantics

<p>For the execution of an <code><a href="./AA-5.7#S0193">exit_statement</a></code>, the <code><a href="./AA-4.5#S0150">condition</a></code>, if present, is first evaluated. If the value of the <code><a href="./AA-4.5#S0150">condition</a></code> is True, or if there is no <code><a href="./AA-4.5#S0150">condition</a></code>, a transfer of control is done to complete the <code>[loop_statement](./AA-5.5#S0178)</code>. If the value of the <code><a href="./AA-4.5#S0150">condition</a></code> is False, no transfer of control takes place. </p>
<p>NOTE 1   Several nested loops can be exited by an <code><a href="./AA-5.7#S0193">exit_statement</a></code> that names the outer loop. </p>

#### Examples

<p><em>Examples of loops with exit statements:</em> </p>
<CodeBlock>
for N in 1 .. Max_Num_Items loop{"\n"}   Get_New_Item(New_Item);{"\n"}   Merge_Item(New_Item, Storage_File);{"\n"}   exit when New_Item = Terminal_Item;{"\n"}end loop;{"\n"}
</CodeBlock>

<CodeBlock>
Main_Cycle:{"\n"}   loop{"\n"}      --  initial statements{"\n"}      exit Main_Cycle when Found;{"\n"}      --  final statements{"\n"}   end loop Main_Cycle;{"\n"}
</CodeBlock>
