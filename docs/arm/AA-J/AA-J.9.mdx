#### Static Semantics

<MarginText>1</MarginText>
<p>For any task subtype T, the following attribute is defined: <br /></p>

<MarginText>2</MarginText>
<dt><br/>T'Storage_Size </dt>
<dl>
<dd>Denotes an implementation-defined value of type <em>universal_integer</em> representing the number of storage elements reserved for a task of the subtype T. <br /></dd>
</dl>
<MarginText>2.a/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<Admonition type="aarm" aarm="note" title="Note: ">
<strong>To be honest: </strong>T'Storage_Size cannot be particularly meaningful in the presence of the specification of the aspect Storage_Size, especially when the expression is dynamic, or depends on a discriminant of the task, because the Storage_Size will be different for different objects of the type. Even without such a specification, the Storage_Size can be different for different objects of the type, and in any case, the value is implementation defined. Hence, it is always implementation defined. <br /></Admonition>

<MarginText>3/3</MarginText>
<MarginInfo items={["AI95-00345-01", "AI05-0229-1"]} />
<dl>
<dd>Storage_Size may be specified for a task first subtype that is not an interface via an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>. When the attribute is specified, the Storage_Size aspect is specified to be the value of the given <code><a href="../AA-4/AA-4.4#S0132">expression</a></code>. <br /></dd>
</dl>
<MarginText>3.a/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>When this attribute is specified with an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>, the associated aspect is set to the <em>value</em> of the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> given in the <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>, rather than the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> itself. This value is therefore the same for all objects of the type; in particular, it is not re-evaluated when objects are created. This is different than when the aspect is specified with an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code> (see <a href="../AA-13/AA-13.3">13.3</a>). <br /></Admonition>


#### Wording Changes from Ada 95

<MarginText>3.b/2</MarginText>
<MarginInfo items={["AI95-00345-01"]} />
<Admonition type="aarm" aarm="note" title="Note: ">
We don't allow specifying Storage_Size on task interfaces. We don't need to mention class-wide task types, because these cannot be a first subtype. <br /></Admonition>
