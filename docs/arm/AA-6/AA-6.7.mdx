# 6.7 Null Procedures

import CodeBlock from "@theme/CodeBlock";
import Admonition from "@theme/Admonition";

<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} A <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> provides a shorthand to declare a procedure with an empty body. </p>

#### Syntax

<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} {"{"}<em>{"AI05-0183-1"}</em>{"}"} <code>null_procedure_declaration</code><a id="S0227"></a><code> ::= </code>   [<code><a href="../AA-8/AA-8.3#S0234">overriding_indicator</a></code>]   <code><a href="../AA-6/AA-6.1#S0197">procedure_specification</a></code> <strong>is</strong> <strong>null</strong>       [<code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>];</p>

#### Legality Rules

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} If a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> is a completion, it shall be the completion of a <code><a href="../AA-6/AA-6.1#S0195">subprogram_declaration</a></code> or <code><a href="../AA-12/AA-12.1#S0311">generic_subprogram_declaration</a></code>. The profile of a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> that completes a declaration shall conform fully to that of the declaration. </p>

#### Static Semantics

<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} {"{"}<em>{"AI05-0177-1"}</em>{"}"} {"{"}<em>{"AI05-0264-1"}</em>{"}"} {"{"}<em>{"AI12-0408-1"}</em>{"}"} A <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> that is not a completion declares a <em>null procedure</em>. A completion is not allowed for a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code>; however, a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> can complete a previous declaration. </p>
<Admonition type="note">
<p><strong>Reason: </strong>There are no null functions because the return value has to be constructed somehow; a function that always raises Program_Error doesn't seem very useful or worth the complication. </p></Admonition>


#### Dynamic Semantics

<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} {"{"}<em>{"AI12-0408-1"}</em>{"}"} The execution of a null procedure is invoked by a subprogram call. For the execution of a subprogram call on a null procedure, or on a procedure completed with a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code>, the execution of the <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code> has no effect. </p>
<Admonition type="note">
<p><strong>Ramification: </strong>Thus, a null procedure is equivalent to the body </p></Admonition>

<CodeBlock>
begin{"\n"}   null;{"\n"}end;{"\n"}
</CodeBlock>

<Admonition type="note">
<p>with the exception that a null procedure can be used in place of a procedure specification. </p></Admonition>

<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} {"{"}<em>{"AI05-0177-1"}</em>{"}"} The elaboration of a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> has no other effect than to establish that the null procedure can be called without failing the Elaboration_Check. </p>

#### Examples

<p>{"{"}<em>{"AI12-0429-1"}</em>{"}"} <em>Example of the declaration of a null procedure:</em> </p>
<CodeBlock>
{"{"}{"AI95-00433-01"}{"}"} {"{"}{"AI12-0440-1"}{"}"} procedure Simplify(Expr : in out Expression) is null; -- see 3.9{"\n"}-- By default, Simplify does nothing, but it can be overridden in extensions of Expression{"\n"}
</CodeBlock>


#### Extensions to Ada 95

<Admonition type="note">
<p>{"{"}<em>{"AI95-00348-01"}</em>{"}"} Null procedures are new. </p></Admonition>


#### Extensions to Ada 2005

<Admonition type="note">
<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} A <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code> can now be a completion.</p></Admonition>

<Admonition type="note">
<p>{"{"}<em>{"AI05-0183-1"}</em>{"}"} An optional <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code> can be used in a <code><a href="../AA-6/AA-6.7#S0227">null_procedure_declaration</a></code>. This is described in 13.1.1. </p></Admonition>


#### Wording Changes from Ada 2012

<Admonition type="note">
<p>{"{"}<em>{"AI12-0408-1"}</em>{"}"} Clarified the term "null procedure" so it matches the meaning expected in 6.1. </p></Admonition>
