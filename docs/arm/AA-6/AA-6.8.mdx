<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} <br />An <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> provides a shorthand to declare a function whose body consists of a single return statement. <br /></p>


#### Syntax

#### Name Resolution Rules

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} {"{"}<em>{"AI12-0157-1"}</em>{"}"} <br />The expected type for the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> or <code><a href="../AA-4/AA-4.3#S0106">aggregate</a></code> of an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> is the result type (see 6.5) of the function. <br /></p>


#### Static Semantics

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} {"{"}<em>{"AI05-0264-1"}</em>{"}"} {"{"}<em>{"AI12-0075-1"}</em>{"}"} {"{"}<em>{"AI12-0157-1"}</em>{"}"} {"{"}<em>{"AI12-0408-1"}</em>{"}"} <br />An <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> that is not a completion declares an <em>expression function</em>. The <em>return expression of an expression function</em> is the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> or <code><a href="../AA-4/AA-4.3#S0106">aggregate</a></code> of the <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>. A completion is not allowed for an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>; however, an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> can complete a previous declaration.<br /></p>

<p>{"{"}<em>{"AI12-0075-1"}</em>{"}"} <br />A <em>potentially static expression</em> is defined in the same way as a static expression except that<br /></p>

<ul>
<li>a name denoting a formal parameter of an expression function is a potentially static expression; and<br /></li>
<li>each use of "static expression" in the definition of "static expression" is replaced with a corresponding use of "potentially static expression" in the definition of "potentially static expression". <br /></li>
</ul>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>These uses occur in the definition of "static expression" in the cases of function calls, type conversions, qualified expressions, membership tests, short circuit control forms, conditional expressions, and parenthesized expressions. <br /></Admonition>

<p>{"{"}<em>{"AI12-0075-1"}</em>{"}"} <br />The following language-defined representation aspect may be specified for an expression function:<br /></p>

<dt><br/>Static</dt>
<dl>
<dd>The type of aspect Static is Boolean. When aspect Static is True for an expression function, the function is a <em>static expression function</em>. If directly specified, the <code><a href="../AA-13/AA-13.1#S0348">aspect_definition</a></code> shall be a static expression.<br /></dd>
</dl>
<Admonition type="aarm" aarm="note" title="Note: ">
<strong>Aspect Description for </strong><strong>Static: </strong>Specifies that an associated expression function can be used in static expressions.<br /></Admonition>

<dt><br/></dt>
<dl>
<dd>The Static value for an inherited function is True if some corresponding primitive function of the parent or progenitor type is a static expression function; otherwise, if not directly specified, the aspect is False. <br /></dd>
</dl>
<p>{"{"}<em>{"AI12-0075-1"}</em>{"}"} <br />[A static expression function is a static function; see 4.9.] <br /></p>


#### Legality Rules

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} <br />If an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> is a completion, it shall be the completion of a <code><a href="../AA-6/AA-6.1#S0195">subprogram_declaration</a></code> or <code><a href="../AA-12/AA-12.1#S0311">generic_subprogram_declaration</a></code>. The profile of an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> that completes a declaration shall conform fully to that of the declaration.<br /></p>

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} {"{"}<em>{"AI12-0157-1"}</em>{"}"} <br />If the result subtype has one or more unconstrained access discriminants, the accessibility level of the anonymous access type of each access discriminant, as determined by the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> or <code><a href="../AA-4/AA-4.3#S0106">aggregate</a></code> of the <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>, shall not be statically deeper than that of the master that elaborated the <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>.<br /></p>

<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>This can only fail if the discriminant is an access to a part of a nonaliased parameter, as there can be no local declarations here. <br /></Admonition>

<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>{"{"}<em>{"AI12-0005-1"}</em>{"}"} <br />We don't need to repeat any of the other Legality Rules for return statements since none of them can fail here: the implicit return statement has to apply to this function (and isn't nested in something), there clearly is a return statement in this function, and the static class-wide accessibility check cannot fail as a tagged type cannot be declared locally in an expression function. <br /></Admonition>

<p>{"{"}<em>{"AI12-0075-1"}</em>{"}"} <br />Aspect Static shall be specified to have the value True only if the associated <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>:<br /></p>

<ul>
<li>is not a completion;<br /></li>
<li>has an <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> that is a potentially static expression;<br /></li>
<li>contains no calls to itself;<br /></li>
<li>each parameter (if any) is of mode <strong>in</strong> and is of a static subtype;<br /></li>
<li>has a result subtype that is a static subtype;<br /></li>
<li>has no applicable precondition or postcondition expression; and<br /></li>
<li>{"{"}<em>{"AI12-0075-1"}</em>{"}"} {"{"}<em>{"AI12-0191-1"}</em>{"}"} <br />for result type <em>R</em>, if the function is a boundary entity for type <em>R</em> (see 7.3.2), no type invariant applies to type <em>R</em>; if <em>R</em> has a component type <em>C</em>, a similar rule applies to <em>C</em>. <br /></li>
</ul>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>Since a string subtype can be static, this allows an expression function of a string type to be static. <br /></Admonition>

#### Dynamic Semantics

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} {"{"}<em>{"AI12-0157-1"}</em>{"}"} {"{"}<em>{"AI12-0408-1"}</em>{"}"} <br />The execution of an expression function is invoked by a subprogram call. For the execution of a subprogram call on an expression function, or on a function completed with a <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code>, the execution of the <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code> executes an implicit function body containing only a <code><a href="../AA-6/AA-6.5#S0222">simple_return_statement</a></code> whose <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> is the return expression of the expression function.<br /></p>

<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>The last sentence effectively means that all of the dynamic wording in 6.5 applies as needed, and we don't have to repeat it here. <br /></Admonition>

<p>{"{"}<em>{"AI05-0177-1"}</em>{"}"} <br />The elaboration of an <code><a href="../AA-6/AA-6.8#S0228">expression_function_declaration</a></code> has no other effect than to establish that the expression function can be called without failing the Elaboration_Check. <br /></p>

#### Extensions to Ada 2012

{"{"}<em>{"AI12-0157-1"}</em>{"}"} <strong><br />Corrigendum:</strong> A <code><a href="../AA-4/AA-4.3#S0106">aggregate</a></code> can directly be the return expression of an expression function. This eliminates the double parentheses that otherwise would be necessary.<br /></Admonition>
