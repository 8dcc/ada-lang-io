#### Static Semantics

<p>The text of a program consists of the texts of one or more <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>s. The text of each <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code> is a sequence of separate <em>lexical elements</em>. Each lexical element is formed from a sequence of characters, and is either a delimiter, an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, a reserved word, a <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code>, a <code><a href="../AA-2/AA-2.5#S0015">character_literal</a></code>, a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>, or a comment. The meaning of a program depends only on the particular sequences of lexical elements that form its <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>s, excluding <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>s.</p>
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} The text of a <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code> is divided into <em>lines</em>. In general, the representation for an end of line is implementation defined. However, a sequence of one or more <code>format_effector</code>s other than the character whose code point is 16#09# (CHARACTER TABULATION) signifies at least one end of line. </p>
<Admonition type="note">
<p><strong>Implementation defined: </strong>The representation for an end of line.</p></Admonition>

<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} [In some cases an explicit <em>separator</em> is required to separate adjacent lexical elements.] A separator is any of a <code>separator_space</code>, a <code>format_effector</code>, or the end of a line, as follows: </p>
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} A <code>separator_space</code> is a separator except within a <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>, a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>, or a <code><a href="../AA-2/AA-2.5#S0015">character_literal</a></code>.</p>
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} The character whose code point is 16#09# (CHARACTER TABULATION) is a separator except within a <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>.</p>
<p>The end of a line is always a separator. </p>
<p>One or more separators are allowed between any two adjacent lexical elements, before the first of each <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>, or after the last. At least one separator is required between an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, a reserved word, or a <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code> and an adjacent <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, reserved word, or <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code>.</p>
<p>{"{"}<em>{"AI05-0079-1"}</em>{"}"} One or more <code>other_format</code> characters are allowed anywhere that a separator is[; any such characters have no effect on the meaning of an Ada program].</p>
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} A <em>delimiter</em> is either one of the following characters: </p>
<p>{"{"}<em>{"AI12-0125-3"}</em>{"}"} {"{"}<em>{"AI12-0212-1"}</em>{"}"} &    '    (    )    *    +    ,        .    /    :    ;    {"<"}    =    {">"}    @    [    ]    |</p>
<p>or one of the following <em>compound delimiters</em> each composed of two adjacent special characters </p>
<p>={">"}    ..    **    :=    /=    {">"}=    {"<"}=    {"<"}{"<"}    {">"}{">"}    {"<"}{">"}</p>
<p>Each of the special characters listed for single character delimiters is a single delimiter except if this character is used as a character of a compound delimiter, or as a character of a <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>, <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>, <code><a href="../AA-2/AA-2.5#S0015">character_literal</a></code>, or <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code>.</p>
