<p>[A <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> is formed by a sequence of graphic characters (possibly none) enclosed between two quotation marks used as string brackets. They are used to represent <code><a href="../AA-6/AA-6.1#S0202">operator_symbol</a></code>s (see 6.1), values of a string type (see 4.2), and array subaggregates (see 4.3.3). ]</p>

#### Syntax

<p><code>string_literal</code><a id="S0016"></a><code> ::= </code>"{"{"}<code><a href="../AA-2/AA-2.6#S0017">string_element</a></code>{"}"}"</p>
<p><code>string_element</code><a id="S0017"></a><code> ::= </code>"" | <em>non_quotation_mark_</em><code>graphic_character</code></p>
<p>A <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> is either a pair of quotation marks (""), or a single <code>graphic_character</code> other than a quotation mark. </p>

#### Static Semantics

<p>The <em>sequence of characters</em> of a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> is formed from the sequence of <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code>s between the bracketing quotation marks, in the given order, with a <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> that is "" becoming a single quotation mark in the sequence of characters, and any other <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> being reproduced in the sequence.</p>
<p>A <em>null string literal</em> is a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> with no <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code>s between the quotation marks.</p>
<Admonition type="aarm" aarm="note" title="Note: ">
<p>NOTE 1   An end of line cannot appear in a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>.</p></Admonition>

<Admonition type="aarm" aarm="note" title="Note: ">
<p>NOTE 2   {"{"}<em>{"AI95-00285-01"}</em>{"}"} No transformation is performed on the sequence of characters of a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>. </p></Admonition>


#### Examples

<p><em>Examples of string literals:</em> </p>
<CodeBlock>
{"{"}{"AI95-00433-01"}{"}"} "Message of the day:"{"\n"}{"\n"}""                    --  a null string literal{"\n"}" "   "A"   """"      --  three string literals of length 1{"\n"}{"\n"}"Characters such as $, %, and {"}"} are allowed in string literals"{"\n"}"Archimedes said """""{"\n"}"Volume of cylinder (r²h) = "{"\n"}
</CodeBlock>


#### Wording Changes from Ada 83

<Admonition type="aarm" aarm="note" title="Note: ">
<p>The wording has been changed to be strictly lexical. No mention is made of string or character values, since <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>s are also used to represent <code><a href="../AA-6/AA-6.1#S0202">operator_symbol</a></code>s, which don't have a defined value.</p></Admonition>

<Admonition type="aarm" aarm="note" title="Note: ">
<p>The syntax is described differently. </p></Admonition>


#### Wording Changes from Ada 95

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} We explicitly say that the characters of a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> should be used as is. In particular, no normalization or folding should be performed on a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>. </p></Admonition>
