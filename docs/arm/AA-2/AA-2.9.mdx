<p>{"{"}<em>{"AI95-00284-02"}</em>{"}"} {"{"}<em>{"AI95-00395-01"}</em>{"}"} {"{"}<em>{"AI05-0091-1"}</em>{"}"} The following are the <em>reserved words</em>. Within a program, some or all of the letters of a reserved word may be in upper case. </p>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<p><strong></strong>Reserved words have special meaning in the syntax. In addition, certain reserved words are used as attribute names.</p></Admonition>

<Admonition type="aarm" aarm="note" title="Note: ">
<p>The syntactic category <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> no longer allows reserved words. We have added the few reserved words that are legal explicitly to the syntax for <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code>. Allowing identifier to include reserved words has been a source of confusion for some users, and differs from the way they are treated in the C and Pascal language definitions.</p></Admonition>

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI12-0005-1"}</em>{"}"} Reserved words are sometimes called <em>keywords</em>  in informal material. </p></Admonition>

#### Incompatibilities With Ada 83

<Admonition type="aarm" aarm="note" title="Note: ">
<p>The following words are not reserved in Ada 83, but are reserved in Ada 95: <strong>abstract</strong>, <strong>aliased</strong>, <strong>protected</strong>, <strong>requeue</strong>, <strong>tagged</strong>, <strong>until</strong>. </p></Admonition>


#### Wording Changes from Ada 83

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI05-0299-1"}</em>{"}"} The subclause entitled "Allowed Replacements of Characters" has been moved to Annex J, "Obsolescent Features". </p></Admonition>


#### Incompatibilities With Ada 95

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI95-00284-02"}</em>{"}"} The following words are not reserved in Ada 95, but are reserved in Ada 2005: <strong>interface</strong>, <strong>overriding</strong>, <strong>synchronized</strong>. A special allowance is made for <strong>pragma</strong> Interface (see J.12). Uses of these words as identifiers will need to be changed, but we do not expect them to be common. </p></Admonition>


#### Wording Changes from Ada 95

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI95-00395-01"}</em>{"}"} The definition of upper case equivalence has been modified to allow identifiers using all of the characters of ISO 10646. This change has no effect on the character sequences that are reserved words, but does make some unusual sequences of characters illegal. </p></Admonition>


#### Incompatibilities With Ada 2005

<Admonition type="aarm" aarm="correction" title="Correction:">
<p>{"{"}<em>{"AI05-0091-1"}</em>{"}"} <strong></strong> Removed <code>other_format</code> characters from reserved words in order to be compatible with the latest Unicode recommendations. This change can only affect programs written for original Ada 2005, and there is little reason to put <code>other_format</code> characters into reserved words in the first place, so there should be very few such programs.</p></Admonition>

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI05-0176-1"}</em>{"}"} The following word is not reserved in Ada 2005, but is reserved in Ada 2012: <strong>some</strong>. Uses of this word as an identifier will need to be changed, but we do not expect them to be common. </p></Admonition>


#### Incompatibilities With Ada 2012

<Admonition type="aarm" aarm="note" title="Note: ">
<p>{"{"}<em>{"AI12-0119-1"}</em>{"}"} The following word is not reserved in Ada 2012, but is reserved in Ada 2022: <strong>parallel</strong>. Uses of this word as an identifier will need to be changed, but we do not expect them to be common. </p></Admonition>
