<p>{"{"}<em>{"AI05-0299-1"}</em>{"}"} <br />[This subclause specifies performance requirements for the <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code>. The rules apply both to <code><a href="../AA-9/AA-9.6#S0268">delay_relative_statement</a></code> and to <code><a href="../AA-9/AA-9.6#S0267">delay_until_statement</a></code>. Similarly, they apply equally to a simple <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> and to one which appears in a <code><a href="../AA-9/AA-9.7#S0274">delay_alternative</a></code>.] <br /></p>


#### Dynamic Semantics

<AnnotatedOnly>
<MarginText>
2
</MarginText>
</AnnotatedOnly>
<p>The effect of the <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> for Real_Time.Time is defined in terms of Real_Time.Clock: <br /></p>

<AnnotatedOnly>
<MarginText>
3
</MarginText>
</AnnotatedOnly>
<ul>
<li>If C1 is a value of Clock read before a task executes a <code><a href="../AA-9/AA-9.6#S0268">delay_relative_statement</a></code> with duration D, and C2 is a value of Clock read after the task resumes execution following that <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code>, then C2  C1 {">"}= D.<br /></li>
<AnnotatedOnly>
<MarginText>
4
</MarginText>
</AnnotatedOnly>
<li>If C is a value of Clock read after a task resumes execution following a <code><a href="../AA-9/AA-9.6#S0267">delay_until_statement</a></code> with Real_Time.Time value T, then C {">"}= T. <br /></li>
<AnnotatedOnly>
<MarginText>
5
</MarginText>
</AnnotatedOnly>
</ul>
<p>A simple <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> with a negative or zero value for the expiration time does not cause the calling task to be blocked; it is nevertheless a potentially blocking operation (see <a href="../AA-9/AA-9.5#Subclause_9.5.1">9.5.1</a>).<br /></p>

<AnnotatedOnly>
<MarginText>
6/3
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0004-1"}</em>{"}"} <br />When a <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> appears in a <code><a href="../AA-9/AA-9.7#S0274">delay_alternative</a></code> of a <code><a href="../AA-9/AA-9.7#S0276">timed_entry_call</a></code> the selection of the entry call is attempted, regardless of the specified expiration time. When a <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> appears in a <code><a href="../AA-9/AA-9.7#S0272">select_alternative</a></code>, and a call is queued on one of the open entries, the selection of that entry call proceeds, regardless of the value of the delay expression. <br /></p>

<AnnotatedOnly>
<MarginText>
6.a
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>The effect of these requirements is that one has to always attempt a rendezvous, regardless of the value of the delay expression. This can be tested by issuing a <code><a href="../AA-9/AA-9.7#S0276">timed_entry_call</a></code> with an expiration time of zero, to an open entry. <br /></Admonition>

#### Metrics

<AnnotatedOnly>
<MarginText>
9
</MarginText>
</AnnotatedOnly>
<p>The implementation shall document the following metrics: <br /></p>

<AnnotatedOnly>
<MarginText>
10
</MarginText>
</AnnotatedOnly>
<ul>
<li>An upper bound on the execution time, in processor clock cycles, of a <code><a href="../AA-9/AA-9.6#S0268">delay_relative_statement</a></code> whose requested value of the delay expression is less than or equal to zero.<br /></li>
<AnnotatedOnly>
<MarginText>
11
</MarginText>
</AnnotatedOnly>
<li>An upper bound on the execution time, in processor clock cycles, of a <code><a href="../AA-9/AA-9.6#S0267">delay_until_statement</a></code> whose requested value of the delay expression is less than or equal to the value of Real_Time.Clock at the time of executing the statement. Similarly, for Calendar.Clock.<br /></li>
<AnnotatedOnly>
<MarginText>
12/5
</MarginText>
</AnnotatedOnly>
<li>{"{"}<em>{"AI12-0445-1"}</em>{"}"} <br />An upper bound on the <em>lateness</em> of a <code><a href="../AA-9/AA-9.6#S0268">delay_relative_statement</a></code>, for a positive value of the delay expression, in a situation where the task has sufficient priority to preempt the processor as soon as it becomes ready, and can proceed without waiting for any other execution resources. The upper bound is expressed as a function of the value of the delay expression. The lateness is obtained by subtracting the value of the delay expression from the <em>actual duration</em>. The actual duration is measured from a point immediately before a task executes the <code><a href="../AA-9/AA-9.6#S0266">delay_statement</a></code> to a point immediately after the task resumes execution following this statement.<br /></li>
<AnnotatedOnly>
<MarginText>
13/5
</MarginText>
</AnnotatedOnly>
<li>{"{"}<em>{"AI12-0445-1"}</em>{"}"} <br />An upper bound on the lateness of a <code><a href="../AA-9/AA-9.6#S0267">delay_until_statement</a></code>, in a situation where the value of the requested expiration time is after the time the task begins executing the statement, the task has sufficient priority to preempt the processor as soon as it becomes ready, and it can proceed without waiting for any other execution resources. The upper bound is expressed as a function of the difference between the requested expiration time and the clock value at the time the statement begins execution. The lateness of a <code><a href="../AA-9/AA-9.6#S0267">delay_until_statement</a></code> is obtained by subtracting the requested expiration time from the real time that the task resumes execution following this statement. <br /></li>
<AnnotatedOnly>
<MarginText>
13.a/2
</MarginText>
</AnnotatedOnly>
</ul>
<Admonition type="aarm" aarm="note" title="Note: ">
<strong>Documentation Requirement: </strong>The metrics for delay statements.<br /></Admonition>
