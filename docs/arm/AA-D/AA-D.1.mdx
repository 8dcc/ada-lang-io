#### Dynamic Semantics

<AnnotatedOnly>
<MarginText>
14/3
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} <br />The Priority aspect has no effect if it is specified for a subprogram other than the main subprogram; the Priority value is not associated with any task.<br /></p>

<AnnotatedOnly>
<MarginText>
15/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0404-1"}</em>{"}"} <br />A <em>task priority</em> is an integer value that indicates a degree of urgency and is the basis for resolving competing demands of tasks for resources. Unless otherwise specified, whenever tasks compete for processors or other implementation-defined resources, the resources are allocated to the task with the highest priority value. The <em>base priority</em> of a task is the priority with which it was created, or to which it was later set by Dynamic_Priorities.Set_Priority (see <a href="../AA-D/AA-D.5">D.5</a>). At all times, a task also has an <em>active priority</em>, which generally is its base priority unless it inherits a priority from other sources. <em>Priority inheritance</em> is the process by which the priority of a task or other entity (for example, a protected object; see <a href="../AA-D/AA-D.3">D.3</a>) is used in the evaluation of another task's active priority. <br /></p>

<AnnotatedOnly>
<MarginText>
15.a
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="implementation-defined" title="Implementation defined: ">
<strong></strong>Implementation-defined execution resources.<br /></Admonition>

<AnnotatedOnly>
<MarginText>
16/3
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} <br />The effect of specifying a Priority or Interrupt_Priority aspect for a protected type or <code><a href="../AA-9/AA-9.4#S0250">single_protected_declaration</a></code> is discussed in <a href="../AA-D/AA-D.3">D.3</a>.<br /></p>

<AnnotatedOnly>
<MarginText>
17/4
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} {"{"}<em>{"AI12-0081-1"}</em>{"}"} <br />The <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> specified for the Priority or Interrupt_Priority aspect of a task type is evaluated each time an object of the task type is created (see <a href="../AA-9/AA-9.1">9.1</a>). For the Priority aspect, the value of the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> is converted to the subtype Priority; for the Interrupt_Priority aspect, this value is converted to the subtype Any_Priority. The priority value is then associated with the task object. <br /></p>

<AnnotatedOnly>
<MarginText>
18/3
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} <br />Likewise, the priority value is associated with the environment task if the aspect is specified for the main subprogram.<br /></p>

<AnnotatedOnly>
<MarginText>
19/3
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} <br />The initial value of a task's base priority is specified by default or by means of a Priority or Interrupt_Priority aspect. [After a task is created, its base priority can be changed only by a call to Dynamic_Priorities.Set_Priority (see <a href="../AA-D/AA-D.5">D.5</a>).] The initial base priority of a task in the absence of an aspect is the base priority of the task that creates it at the time of creation (see <a href="../AA-9/AA-9.1">9.1</a>). If the aspect Priority is not specified for the main subprogram, the initial base priority of the environment task is System.Default_Priority. [The task's active priority is used when the task competes for processors. Similarly, the task's active priority is used to determine the task's position in any queue when Priority_Queuing is specified (see <a href="../AA-D/AA-D.4">D.4</a>).]<br /></p>

<AnnotatedOnly>
<MarginText>
20/2
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI95-00357-01"}</em>{"}"} <br />At any time, the active priority of a task is the maximum of all the priorities the task is inheriting at that instant. For a task that is not held (see <a href="../AA-D/AA-D.11">D.11</a>), its base priority is a source of priority inheritance unless otherwise specified for a particular task dispatching policy. Other sources of priority inheritance are specified under the following conditions: <br /></p>

<AnnotatedOnly>
<MarginText>

</MarginText>

<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>{"{"}<em>{"AI12-0005-1"}</em>{"}"} {"{"}<em>{"AI12-0276-1"}</em>{"}"} <br />Priority inheritance is needed for FIFO_Spinning to ensure that lower priority tasks that initiate spin waiting earlier than other higher priority tasks continue to spin to ensure that they can be granted the resource when it becomes available in order to support FIFO ordering. Note that this rule only matters when tasks that can initiate a protected action on an object P can be on a different processor than P. In particular, this rule does not matter on a monoprocessor. <br /></Admonition>

<MarginText>

</MarginText>
