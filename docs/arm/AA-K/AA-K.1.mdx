# K.1 Language-Defined Aspects

<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} {"{"}<em>{"AI05-0299-1"}</em>{"}"} <br />This subclause summarizes the definitions given elsewhere of the language-defined aspects. Aspects are properties of entities that can be specified by the Ada program; unless otherwise specified below, aspects can be specified using an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>.<br /></p>

<dt><br/>Address</dt>
<dl>
<dd>Machine address of an entity. See 13.3.<br /></dd>
<dt><br/>Aggregate</dt>
<dd>Mechanism to define user-defined aggregates. See 4.3.5.<br /></dd>
<dt><br/>Alignment (object)</dt>
<dd>Alignment of an object. See 13.3.<br /></dd>
<dt><br/>Alignment (subtype)</dt>
<dd>Alignment of a subtype. See 13.3.<br /></dd>
<dt><br/>All_Calls_Remote</dt>
<dd>All indirect or dispatching remote subprogram calls, and all direct remote subprogram calls, should use the Partition Communication Subsystem. See E.2.3.<br /></dd>
<dt><br/>Allows_Exit</dt>
<dd>An indication of whether a subprogram will operate correctly for arbitrary transfers of control. See 5.5.3.<br /></dd>
<dt><br/>Asynchronous</dt>
<dd>Remote procedure calls are asynchronous; the caller continues without waiting for the call to return. See E.4.1.<br /></dd>
<dt><br/>Atomic</dt>
<dd>Declare that a type, object, or component is atomic. See C.6.<br /></dd>
<dt><br/>Atomic_Components</dt>
<dd>Declare that the components of an array type or object are atomic. See C.6.<br /></dd>
<dt><br/>Attach_Handler</dt>
<dd>Protected procedure is attached to an interrupt. See C.3.1.<br /></dd>
<dt><br/>Bit_Order</dt>
<dd>Order of bit numbering in a <code><a href="../AA-13/AA-13.5#S0352">record_representation_clause</a></code>. See 13.5.3.<br /></dd>
<dt><br/>Coding</dt>
<dd>Internal representation of enumeration literals. Specified by an <code><a href="../AA-13/AA-13.4#S0350">enumeration_representation_clause</a></code>, not by an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>. See 13.4.<br /></dd>
<dt><br/>Component_Size</dt>
<dd>Size in bits of a component of an array type. See 13.3.<br /></dd>
<dt><br/>Constant_Indexing</dt>
<dd>Defines function(s) to implement user-defined <code><a href="../AA-4/AA-4.1#S0096">indexed_component</a></code>s. See 4.1.6.<br /></dd>
<dt><br/>Convention</dt>
<dd>Calling convention or other convention used for interfacing to other languages. See B.1.<br /></dd>
<dt><br/>CPU</dt>
<dd>Processor on which a given task, or calling task for a protected operation, should run. See D.16.<br /></dd>
<dt><br/>Default_Component_Value</dt>
<dd>Default value for the components of an array-of-scalar subtype. See 3.6.<br /></dd>
<dt><br/>Default_Initial_Condition</dt>
<dd>A condition that will hold true after the default initialization of an object. See 7.3.3.<br /></dd>
<dt><br/>Default_Iterator</dt>
<dd>Default iterator to be used in <strong>for</strong> loops. See 5.5.1.<br /></dd>
<dt><br/>Default_Storage_Pool</dt>
<dd>Default storage pool for a generic instance. See 13.11.3.<br /></dd>
<dt><br/>Default_Value</dt>
<dd>Default value for a scalar subtype. See 3.5.<br /></dd>
<dt><br/>Discard_Names</dt>
<dd>Requests a reduction in storage for names associated with an entity. See C.5.<br /></dd>
<dt><br/>Dispatching</dt>
<dd>Generic formal parameters used in the implementation of an entity. See H.7.1.<br /></dd>
<dt><br/>Dispatching_Domain</dt>
<dd>Domain (group of processors) on which a given task should run. See D.16.1.<br /></dd>
<dt><br/>Dynamic_Predicate</dt>
<dd>Condition that will hold true for objects of a given subtype; the subtype is not static. See 3.2.4.<br /></dd>
<dt><br/>Elaborate_Body</dt>
<dd>A given package will have a body, and that body is elaborated immediately after the declaration. See 10.2.1.<br /></dd>
<dt><br/>Exclusive_Functions</dt>
<dd>Specifies mutual exclusion behavior of protected functions in a protected type. See 9.5.1.<br /></dd>
<dt><br/>Export</dt>
<dd>Entity is exported to another language. See B.1.<br /></dd>
<dt><br/>External_Name</dt>
<dd>Name used to identify an imported or exported entity. See B.1.<br /></dd>
<dt><br/>External_Tag</dt>
<dd>Unique identifier for a tagged type in streams. See 13.3.<br /></dd>
<dt><br/>Full_Access_Only</dt>
<dd>Declare that a volatile type, object, or component is full access. See C.6.<br /></dd>
<dt><br/>Global</dt>
<dd>Global object usage contract. See 6.1.2.<br /></dd>
<dt><br/>Global'Class</dt>
<dd>Global object usage contract inherited on derivation. See 6.1.2.<br /></dd>
<dt><br/>Implicit_Dereference</dt>
<dd>Mechanism for user-defined implicit .<strong>all</strong>. See 4.1.5.<br /></dd>
<dt><br/>Import</dt>
<dd>Entity is imported from another language. See B.1.<br /></dd>
<dt><br/>Independent</dt>
<dd>Declare that a type, object, or component is independently addressable. See C.6.<br /></dd>
<dt><br/>Independent_Components</dt>
<dd>Declare that the components of an array or record type, or an array object, are independently addressable. See C.6.<br /></dd>
<dt><br/>Inline</dt>
<dd>For efficiency, Inline calls are requested for a subprogram. See 6.3.2.<br /></dd>
<dt><br/>Input</dt>
<dd>Function to read a value from a stream for a given type, including any bounds and discriminants. See 13.13.2.<br /></dd>
<dt><br/>Input'Class</dt>
<dd>Function to read a value from a stream for a the class-wide type associated with a given type, including any bounds and discriminants. See 13.13.2.<br /></dd>
<dt><br/>Integer_Literal</dt>
<dd>Defines a function to implement user-defined integer literals. See 4.2.1.<br /></dd>
<dt><br/>Interrupt_Handler</dt>
<dd>Protected procedure may be attached to interrupts. See C.3.1.<br /></dd>
<dt><br/>Interrupt_Priority</dt>
<dd>Priority of a task object or type, or priority of a protected object or type; the priority is in the interrupt range. See D.1.<br /></dd>
<dt><br/>Iterator_Element</dt>
<dd>Element type to be used for user-defined iterators. See 5.5.1.<br /></dd>
<dt><br/>Iterator_View</dt>
<dd>An alternative type to used for container element iterators. See 5.5.1.<br /></dd>
<dt><br/>Layout (record)</dt>
<dd>Layout of record components. Specified by a <code><a href="../AA-13/AA-13.5#S0352">record_representation_clause</a></code>, not by an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>. See 13.5.1.<br /></dd>
<dt><br/>Link_Name</dt>
<dd>Linker symbol used to identify an imported or exported entity. See B.1.<br /></dd>
<dt><br/>Machine_Radix</dt>
<dd>Radix (2 or 10) that is used to represent a decimal fixed point type. See F.1.<br /></dd>
<dt><br/>Max_Entry_Queue_Length</dt>
<dd>The maximum entry queue length for a task type, protected type, or entry. See D.4.<br /></dd>
<dt><br/>No_Controlled_Parts</dt>
<dd>A specification that a type and its descendants do not have controlled parts. See H.4.1.<br /></dd>
<dt><br/>No_Return</dt>
<dd>A subprogram will not return normally. See 6.5.1.<br /></dd>
<dt><br/>Nonblocking</dt>
<dd>Specifies that an associated subprogram does not block. See 9.5.<br /></dd>
<dt><br/>Output</dt>
<dd>Procedure to write a value to a stream for a given type, including any bounds and discriminants. See 13.13.2.<br /></dd>
<dt><br/>Output'Class</dt>
<dd>Procedure to write a value to a stream for a the class-wide type associated with a given type, including any bounds and discriminants. See 13.13.2.<br /></dd>
<dt><br/>Pack</dt>
<dd>Minimize storage when laying out records and arrays. See 13.2.<br /></dd>
<dt><br/>Parallel_Calls</dt>
<dd>Specifies whether a given subprogram is expected to be called in parallel. See 9.10.1.<br /></dd>
<dt><br/>Parallel_Iterator</dt>
<dd>An indication of whether a subprogram may use multiple threads of control to invoke a loop body procedure. See 5.5.3.<br /></dd>
<dt><br/>Post</dt>
<dd>Postcondition; a condition that will hold true after a call. See 6.1.1.<br /></dd>
<dt><br/>Post'Class</dt>
<dd>Postcondition that applies to corresponding subprograms of descendant types. See 6.1.1.<br /></dd>
<dt><br/>Pre</dt>
<dd>Precondition; a condition that is expected to hold true before a call. See 6.1.1.<br /></dd>
<dt><br/>Pre'Class</dt>
<dd>Precondition that applies to corresponding subprograms of descendant types. See 6.1.1.<br /></dd>
<dt><br/>Predicate_Failure</dt>
<dd>Action to be performed when a predicate check fails. See 3.2.4.<br /></dd>
<dt><br/>Preelaborable_Initialization</dt>
<dd>Declares that a type has preelaborable initialization. See 10.2.1.<br /></dd>
<dt><br/>Preelaborate</dt>
<dd>Code execution during elaboration is avoided for a given package. See 10.2.1.<br /></dd>
<dt><br/>Priority</dt>
<dd>Priority of a task object or type, or priority of a protected object or type; the priority is not in the interrupt range. See D.1.<br /></dd>
<dt><br/>Pure</dt>
<dd>Side effects are avoided in the subprograms of a given package. See 10.2.1.<br /></dd>
<dt><br/>Put_Image</dt>
<dd>Procedure to define the image of a given type. See 4.10.<br /></dd>
<dt><br/>Read</dt>
<dd>Procedure to read a value from a stream for a given type. See 13.13.2.<br /></dd>
<dt><br/>Read'Class</dt>
<dd>Procedure to read a value from a stream for the class-wide type associated with a given type. See 13.13.2.<br /></dd>
<dt><br/>Real_Literal</dt>
<dd>Defines a function or functions to implement user-defined real literals. See 4.2.1.<br /></dd>
<dt><br/>Record layout</dt>
<dd>See Layout. See 13.5.1.<br /></dd>
<dt><br/>Relative_Deadline</dt>
<dd>Task or protected type parameter used in Earliest Deadline First Dispatching. See D.2.6.<br /></dd>
<dt><br/>Remote_Call_Interface</dt>
<dd>Subprograms in a given package may be used in remote procedure calls. See E.2.3.<br /></dd>
<dt><br/>Remote_Types</dt>
<dd>Types in a given package may be used in remote procedure calls. See E.2.2.<br /></dd>
<dt><br/>Shared_Passive</dt>
<dd>A given package is used to represent shared memory in a distributed system. See E.2.1.<br /></dd>
<dt><br/>Size (object)</dt>
<dd>Size in bits of an object. See 13.3.<br /></dd>
<dt><br/>Size (subtype)</dt>
<dd>Size in bits of a subtype. See 13.3.<br /></dd>
<dt><br/>Small</dt>
<dd>Scale factor for a fixed point type. See 3.5.10.<br /></dd>
<dt><br/>Stable_Properties</dt>
<dd>A list of functions describing characteristics that usually are unchanged by primitive operations of the type or an individual primitive subprogram. See 7.3.4.<br /></dd>
<dt><br/>Stable_Properties'Class</dt>
<dd>A list of functions describing characteristics that usually are unchanged by primitive operations of a class of types or a primitive subprogram for such a class. See 7.3.4.<br /></dd>
<dt><br/>Static</dt>
<dd>Specifies that an associated expression function can be used in static expressions. See 6.8.<br /></dd>
<dt><br/>Static_Predicate</dt>
<dd>Condition that will hold true for objects of a given subtype; the subtype may be static. See 3.2.4.<br /></dd>
<dt><br/>Storage_Pool</dt>
<dd>Pool of memory from which <strong>new</strong> will allocate for a given access type. See 13.11.<br /></dd>
<dt><br/>Storage_Size (access)</dt>
<dd>Sets memory size for allocations for an access type. See 13.11.<br /></dd>
<dt><br/>Storage_Size (task)</dt>
<dd>Size in storage elements reserved for a task type or single task object. See 13.3.<br /></dd>
<dt><br/>Stream_Size</dt>
<dd>Size in bits used to represent elementary objects in a stream. See 13.13.2.<br /></dd>
<dt><br/>String_Literal</dt>
<dd>Defines a function to implement user-defined string literals. See 4.2.1.<br /></dd>
<dt><br/>Synchronization</dt>
<dd>Defines whether a given primitive operation of a synchronized interface will be implemented by an entry or protected procedure. See 9.5.<br /></dd>
<dt><br/>Type_Invariant</dt>
<dd>A condition that will hold true for all objects of a type. See 7.3.2.<br /></dd>
<dt><br/>Type_Invariant'Class</dt>
<dd>A condition that will hold true for all objects in a class of types. See 7.3.2.<br /></dd>
<dt><br/>Unchecked_Union</dt>
<dd>Type is used to interface to a C union type. See B.3.3.<br /></dd>
<dt><br/>Use_Formal</dt>
<dd>Generic formal parameters used in the implementation of an entity. See H.7.1.<br /></dd>
<dt><br/>Variable_Indexing</dt>
<dd>Defines function(s) to implement user-defined <code><a href="../AA-4/AA-4.1#S0096">indexed_component</a></code>s. See 4.1.6.<br /></dd>
<dt><br/>Volatile</dt>
<dd>Declare that a type, object, or component is volatile. See C.6.<br /></dd>
<dt><br/>Volatile_Components</dt>
<dd>Declare that the components of an array type or object are volatile. See C.6.<br /></dd>
<dt><br/>Write</dt>
<dd>Procedure to write a value to a stream for a given type. See 13.13.2.<br /></dd>
<dt><br/>Write'Class</dt>
<dd>Procedure to write a value to a stream for a the class-wide type associated with a given type. See 13.13.2.<br /></dd>
<dt><br/>Yield</dt>
<dd>Ensures that a callable entity includes a task dispatching point. See D.2.1.<br /></dd>
</dl>
